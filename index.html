Concepts of the Web

How the web works

HTTP

The web consists of computers that talk to each other over the Internet. The computers talk to each other with a protocol called http. The end user's computer runs a web browser which is responsible for requesting documents from servers, which store them, and displaying the contents to the user. The user's computer sends a document request to the server using the http protocol. When the server receives the request, it looks for the document and returns it to the user also using the http protocol. A good example video of how this works can be found here: This video explains the process.

HTML

The primary document type that makes up the web is called a HTML document. HTML stands for Hypertext Markup Language and is used to control the way the document is displayed to the user. The user's computer reads the document and displays it as described by the HTML code. The HTML document consists of content to be displayed as well as instructions on how to display the content. A good overview of HTML documents can be seen in this video.

Tags and Elements

HTML documents consists of various parts, but they primarily consist of instructions on how to format the content, called tags and the content itelf. This combination of tags and content is often referred to as an Element. A tag is used to tell the browser how to treat the contents as well as providing a means to link to external content not contained in the document itself. Tags and elements are explained in more detail in this video. Referring to external content is a powerful way to create interesting and dynamic pages that can display videos or play music.

Computers are stupid

Computers only follow intstructions as provided to them. They intepret the instructions literally, so even very small mistakes in those instructions can cause unexpected behavior.

Inline vs Block Elements

HTML elements have two primary distinguishing characteristics that allow certain degrees of control. An Inline element is basic item that is usually rendered similar to ordinary text. A Block element, however, forms a sort of invisible block around the content such that it can be formatted in various ways.
